# portal-main-journal

一个用于归档和发布多个独立 HTML 页面的仓库。这里存放着各类静态网页资源，作为内容集合与历史记录。

## 项目简介

本仓库的主要目的是系统性地整理和保存零散的 HTML 页面，避免原始文件散落丢失。所有页面均为独立文件，不依赖特定后端服务或数据库，可直接在浏览器中打开查看。

适合存放：
- 临时生成的报告页
- 工具或演示界面
- 信息展示页面
- 其他需要保留的静态网页

## 目录说明

```
portal-main-journal/
├── pages/          # 主要页面存放目录
├── assets/         # 共用资源文件（CSS、JS、图片等）
├── archive/        # 已归档的旧版本或不再活跃的页面
└── index.html      # 仓库首页导航（如有）
```

各目录下按页面功能或时间进行组织，具体结构以实际提交为准。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，包含必要的内联样式或外部资源引用。
- 资源文件（如样式表、脚本、图片）统一放置在 `assets/` 目录下，便于维护。
- 归档页面移入 `archive/` 目录，保留原始文件名，方便追溯。
- 不建议在页面中嵌入敏感信息或动态外部依赖，以保证长期可访问性。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并更新根目录的 `index.html`（如有）。
- 更新页面：直接覆盖原文件，如需保留旧版本，请先移入 `archive/`。
- 删除页面：确认无用后，从仓库中移除，或移至 `archive/` 作为备份。
- 欢迎提交 Pull Request 贡献新页面或改进现有内容，请保持页面代码整洁，并附带简要说明。
